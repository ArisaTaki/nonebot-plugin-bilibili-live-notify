# 开发与本地测试

这份文档面向开发者，不面向插件最终用户。

## 1. 自动化测试

```bash
python -m pip install -e ".[test]"
pytest -q
```

当前测试覆盖：

- 存储层读写与退订清理
- 命令主流程
- 代理协议与状态归一化逻辑

## 2. 本地代理 smoke test

先启动代理服务：

```bash
uvicorn main:app --app-dir bilibili-proxy --host 127.0.0.1 --port 8000
```

再执行：

```bash
bash scripts/smoke_proxy.sh 711308
```

这一步验证的是代理服务是否真的能返回插件期望的协议。

## 3. NoneBot 本地联调

NoneBot 没有一个“官方云沙盒”来替代所有真实运行场景。

通常分成两层：

- 自动化测试：用 `pytest` + `nonebug` 验证 matcher、命令流和部分交互逻辑
- 真实联调：在本地或服务器启动机器人，用真实适配器连接协议端测试

对于这个插件，自动化测试已经能覆盖主要业务逻辑，但最终仍建议做一次真实联调，原因是：

- 需要验证 OneBot V11 适配器实际发群消息是否正常
- 需要验证图片、`@`、权限和群消息上下文是否符合预期
- 需要确认代理服务到机器人运行环境的网络访问没问题

## 4. 是否必须用真实 QQ 账号

严格说，不一定非要你自己的主账号，但需要一个真实可用的 QQ 机器人运行环境来做最终联调。

也就是说：

- 单元测试和 matcher 测试不需要真实 QQ
- 最终上线前的端到端验证通常仍需要真实 OneBot 环境

如果你本地已经有 napcat、lagrange、go-cqhttp 或其他 OneBot V11 环境，那么直接拿它做最后一轮联调即可。

## 5. 发布前最低检查

建议至少完成以下几项：

1. `pytest -q`
2. `bash scripts/smoke_proxy.sh 711308`
3. 在真实 OneBot V11 环境里测试一次 `订阅直播` 和 `直播订阅列表`
4. `python -m build`
